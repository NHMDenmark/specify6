Specify Software Project


Release Notes
Specify 6.6.06
2 October 2017

Database Schema Changes:

  1. Added several data fields to accommodate DNA Extract and DNA Sequence metadata as defined by the Global Genome Biodiversity Network (GGBN, (http://www.ggbn.org/) ) and used in TDWG Darwin Core GGBN Registered Extensions. Fields added are from GGBN Vocabularies for: Amplification, Material Sample, Permit, and Preparation.
  2. Added several fields for NIH SRA accession numbers to enable linking sample and sequence data from next generation methods deposited in NIH SRA databases to the corresponding DNA Sequence record in Specify. This allows http link-outs to molecular data at NIH, sequenced from specimens/tissues/extracts in the collection.
  3. To make Specify compliant with the TDWG GGBN Material Sample DwC extension (https://terms.tdwg.org/wiki/GGBN_Data_Standard) for DNA/RNA extract (sample) metadata, a data table “Material Sample” was added to the database. The Preparation table describes physical objects, like skin, skeleton, seeds, and tissue. Material Samples are derived chemical extracts from Preparations. The DNA Sequence table describes molecular information derived (e.g. sequenced) from a Material Sample (a new relationship with this release) or from a Collection Object (existing Specify relationship). Specify sites using the DNA Sequence table prior to this release (as a child of Collection Object) should contact the Specify Project to discuss the possibility of using the Material Sample--DNA Sequence relationship in its place. More information is available on the Specify Software Project web site about the new Material Sample table, its relationship to the DNA Sequence table, and the relevance of those two changes for managing data for extracts and molecular sequences.
  4. Existing TDWG Audubon Core standard fields were supplemented with additional commonly used Audubon Core vocabulary terms (fields) for describing and exporting multimedia objects (Attachments in Specify).
  5. Added AdminContact (Agent) field to the Collection table for identifying collections in feedback stats.
  6. Added long integer fields to Taxon table to allow for assignment of numeric codes to taxa for sorting and ordering taxon names in query results.
  7. Added text fields to Borrow Materials table, to allow extended description of borrowed materials.
  8. Added text and logical (yes/no) fields to the Collection Object Attributes table.
  9. Added 16 logical fields to the Taxon table.
  10. Added 4 memo fields, 11 string, and 2 integer fields to the Taxon table.
  11. Expanded AltCatalogNumber to 64 chars.
  12. Added Geocoord error estimate fields to the Workbench Row table.
  13. Increased the length of picklist value and title fields from 64 to 128 chars.
